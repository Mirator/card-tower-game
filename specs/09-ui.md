# UI

## Game Screen Specification (based on reference UI)

This section defines the exact layout and behavior of the main gameplay screen inspired by the provided screenshot.

## Layout overview

The screen is divided into 4 main zones:

- Top center (game info)
- Left panel (Player A)
- Right panel (Player B)
- Bottom area (player hand + actions)

The background is a static image (e.g., forest/anthill) with UI layered on top.

## 1. Top center area

Contains shared game state:

- Deck (face-down card)
- Opponent identity panel
- Name
- Tower value (large, e.g., "hrad: 22")
- Optional: turn indicator (small icon or glow)

Behavior:

- Deck animates when drawing cards
- Active player highlighted

## 2. Left player panel (Player A)

Vertical info panel aligned left.

Sections (top -> bottom)

Header

- Player name (e.g., "Cerni")
- Color-coded background (blue/red/green variant)

Generators + resources

Three stacked blocks:

- Builders (Quarry / Bricks)
  - Label: "Stavitele"
  - Resource: "Cihly"
  - Shows:
    - generator value (big number)
    - resource amount (smaller number)
- Soldiers (Barracks / Weapons)
  - "Vojaci" / "Zbrane"
- Mages (Magic / Crystals)
  - "Magove" / "Krystaly"

Each block:

- Has icon (hammer, sword, star)
- Color-coded background

Tower + Wall (bottom block)

- Tower ("Hrad")
- Wall ("Hradba")
- Large numbers
- Visual icon (castle)

## 3. Right player panel (Player B)

Mirrored version of left panel:

- Same structure
- Right-aligned
- Color-coded differently

## 4. Center gameplay area

Contains visual towers:

- Left tower = Player A
- Right tower = Player B

Tower visualization:

- Multi-level colored windows (represent progress visually)
- Height does NOT scale dynamically (cosmetic only)

Optional enhancement:

- slight animation on damage (shake)
- glow on heal

## 5. Bottom area (player hand)

Horizontal card row.

Card layout

Each card shows:

- Name (top)
- Cost (top-right or badge)
- Effect text (short)
- Type color:
  - Brick = red/brown
  - Weapon = green
  - Crystal = blue

Example (from screenshot):

- "Zed"
- "Caruj cihly"
- "Kletba"
- "Smrtka"

Interaction

Desktop:

- Click = play
- Right click or button = discard

Mobile:

- Tap = select
- Swipe up = play
- Swipe down = discard

States:

- Affordable -> full color
- Not affordable -> dimmed

## 6. Visual feedback

Required feedback elements:

- Damage numbers (floating)
- Resource gain popups (+X)
- Generator changes
- Card played animation (move to center, resolve, fade)

## 7. Turn flow indicators

- Active player highlight (glow on panel)
- "Your turn" label
- Optional timer (future)

## 8. Phaser implementation mapping

Scene structure

`GameScene` contains:

- Background (image)
- LeftPanelContainer
- RightPanelContainer
- TowerContainer (center)
- HandContainer (bottom)
- OverlayContainer (effects/logs)

UI components (Phaser Containers)

- PlayerPanel
- ResourceBlock
- TowerDisplay
- CardView
- HandView

Z-order

- Background
- Towers
- Panels
- Cards
- Effects (top layer)
