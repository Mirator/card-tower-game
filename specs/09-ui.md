# UI

## Game Screen Specification (based on reference UI)

This section defines the gameplay screen layout. The current direction keeps the readable reference layout, but pushes the live game toward a stronger tower-duel stage with lower chrome and clearer turn feedback.

## Layout overview

The screen is divided into 4 main zones:

- Slim top turn bar
- Left panel (Player A resources)
- Right panel (Player B resources)
- Center battlefield + bottom player cockpit

The background is a static image (e.g., forest/anthill) with UI layered on top.

## 1. Top center area

Contains shared game state without duplicating the full opponent panel:

- Deck (face-down card)
- Current turn headline
- Short enemy tower/wall summary
- Win goal and player deck count
- Turn indicator chips

Behavior:

- Deck animates when drawing cards
- Active player highlighted
- Enemy reveal temporarily changes the headline before effects resolve

## 2. Left player panel (Player A)

Vertical info panel aligned left.

Sections (top -> bottom)

Header

- Player name (e.g., "Cerni")
- Color-coded background (blue/red/green variant)

Generators + resources

Three stacked blocks:

- Builders (Quarry / Bricks)
  - Label: "Stavitele"
  - Resource: "Cihly"
  - Shows:
    - generator value (big number)
    - resource amount (smaller number)
- Soldiers (Barracks / Weapons)
  - "Vojaci" / "Zbrane"
- Mages (Magic / Crystals)
  - "Magove" / "Krystaly"

Each block:

- Has icon (hammer, sword, star)
- Color-coded background

Tower + Wall (bottom block)

- Tower ("Hrad")
- Wall ("Hradba")
- Large numbers
- Visual icon (castle)

## 3. Right player panel (Player B)

Mirrored version of left panel:

- Same structure
- Right-aligned
- Color-coded differently

## 4. Center battlefield

Contains the main duel stage:

- Left tower = Player A
- Right tower = Player B
- Shared battlefield platform and attack lane
- Tower progress meter toward the win condition
- Wall/shield visual around each tower
- Danger glow when a tower is low

Tower visualization:

- Multi-level colored windows (represent progress visually)
- Height does NOT scale dynamically (cosmetic only), but progress bars and text show tower pressure

Optional enhancement:

- Card travel toward the affected tower or resource panel
- Impact burst on resolution
- Damage shake and heal glow

## 5. Bottom area (player cockpit)

The bottom area is the player's main cockpit: large hand cards, selected-card preview, compact battle feed, and minimal controls hint.

Card layout

Each card shows:

- Name (top)
- Cost (top-right or badge)
- Effect text (short)
- Type color:
  - Brick = red/brown
  - Weapon = green
  - Crystal = blue

Example (from screenshot):

- "Zed"
- "Caruj cihly"
- "Kletba"
- "Smrtka"

Interaction

Desktop:

- Click = play
- Right click or button = discard
- Hover = inspect and lift card

Mobile:

- Tap = select
- Swipe up = play
- Swipe down = discard

States:

- Affordable -> full color
- Not affordable -> dimmed
- Selected -> lifted with stronger border

## 6. Visual feedback

Required feedback elements:

- Damage numbers (floating)
- Resource gain popups (+X)
- Generator changes
- Enemy card reveal before effects resolve
- Card played animation (card travels toward affected tower/resource panel, impact burst, then fade)
- Battle feed with recent events

## 7. Turn flow indicators

- Active player highlight (glow on panel)
- "Your turn" label
- No turn countdown/timer in v1; pacing comes from one-card turns and immediate AI response
- Top bar uses concise turn instructions instead of persistent long control text

## 8. Phaser implementation mapping

Scene structure

`GameScene` contains:

- Background (image)
- LeftPanelContainer
- RightPanelContainer
- TowerContainer (center)
- HandContainer (bottom)
- OverlayContainer (effects/logs)

UI components (Phaser Containers)

- PlayerPanel
- ResourceBlock
- TowerDisplay
- CardView
- HandView

Z-order

- Background
- Towers
- Panels
- Cards
- Effects (top layer)

## 9. Differences vs original reference

Planned improvements:

- Clear typography (no pixel blur)
- Consistent spacing grid
- Unified color system
- Better readability of numbers
- Hover/tap previews
- Animation feedback
- Stronger battlefield hierarchy
- Larger tactile cards
- Reduced always-on instruction text
