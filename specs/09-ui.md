# UI

## Game Screen Specification

The implemented game uses a readable tower-duel stage with low chrome, clear turn feedback, a selected-card command panel, and responsive desktop/mobile layouts.

## Layout overview

The screen is divided into 4 main zones:

- Slim top turn bar
- Left panel (Player A resources)
- Right panel (Player B resources)
- Center battlefield + bottom player cockpit

The background is drawn in Phaser as a static atmospheric battlefield scene with UI layered on top.

## 1. Top center area

Contains shared game state without duplicating the full opponent panel:

- Deck (face-down card)
- Current turn headline
- Short enemy tower/wall summary
- Win goal and player deck count
- Turn indicator chips

Behavior:

- Deck animates when cards are drawn
- Active player is highlighted
- Enemy reveal temporarily changes the headline before effects resolve
- No turn timer is shown

## 2. Left player panel (Player A)

Vertical info panel aligned left on desktop, compacted on narrow screens.

Sections (top -> bottom):

- Header: `Player A`
- Builders / Bricks block: Quarry gain shown as `+X`, Bricks amount shown separately
- Soldiers / Weapons block: Barracks gain shown as `+X`, Weapons amount shown separately
- Mages / Crystals block: Magic gain shown as `+X`, Crystals amount shown separately
- Tower / Wall block with large numbers

## 3. Right player panel (Player B)

Mirrored version of the left panel:

- Header: `Player B`
- Same generator/resource order as Player A
- Active turn glow when the AI is acting
- Tower and wall values remain visible

## 4. Center battlefield

Contains the main duel stage:

- Left tower = Player A
- Right tower = Player B
- Shared battlefield platform and attack lane
- Tower progress meter toward the win condition
- Wall/shield visual around each tower
- Danger glow when a tower is low

Tower visualization:

- Multi-level colored windows represent tower pressure visually
- Height does not scale dynamically; progress bars and text show the real tower value
- Played cards travel toward the affected tower or resource panel before resolving
- Impacts use bursts, shakes, floating numbers, and heal/resource feedback

## 5. Bottom area (player cockpit)

The bottom area is the player's main cockpit: large hand cards, selected-card preview, explicit action controls, compact battle feed, and minimal controls hint.

Card layout:

- Name
- Cost badge
- Short effect text
- Type color:
  - Brick = red/brown
  - Weapon = green
  - Crystal = blue

Interaction:

Desktop:

- Left click a card = play it if possible
- Right click a card = discard it
- Hover = inspect and lift card
- Enter = play selected card
- Backspace/Delete = discard selected card
- Play and Discard buttons remain available for the selected card

Mobile:

- Tap = select
- Swipe up = play
- Swipe down = discard
- Visible Play / Discard buttons remain available for the selected card

States:

- Affordable -> full color
- Not affordable -> dimmed
- Selected -> lifted with stronger border

Selected-card command panel:

- Shows the selected card name
- Shows whether the card is ready or how many resources are missing
- Provides explicit Play and Discard buttons
- Disables Play when the card is unaffordable or it is not the player's action window

## 6. Visual feedback

Required feedback elements:

- Damage numbers (floating)
- Resource gain popups (+X)
- Generator changes
- Enemy card reveal before AI effects resolve
- Card played animation (card travels toward affected tower/resource panel, impact burst, then fade)
- Battle feed with recent events
- Selected card impact preview before committing

## 7. Turn flow indicators

- Active player highlight (glow on panel)
- `Your turn`, `Opponent turn`, and match-over labels
- No turn timer in v1; pacing comes from one-card turns and immediate AI response
- Top bar uses concise turn instructions instead of persistent long control text

## 8. Phaser implementation mapping

`GameScene` contains:

- BackgroundContainer
- TopCenterContainer
- LeftPanelContainer
- RightPanelContainer
- TowerContainer
- HandContainer
- OverlayContainer
- EndOverlay

UI component groups:

- Player panel
- Resource block
- Tower visual
- Card view
- Hand view
- Selected-card action panel
- Battle feed
- Enemy card reveal

Z-order:

- Background
- Towers
- Panels
- Cards
- Effects
- End overlay

## 9. Implemented improvements vs original reference

- Clear typography
- Consistent spacing grid
- Unified color system
- Better readability of numbers
- Hover/tap previews
- Animation feedback
- Stronger battlefield hierarchy
- Larger tactile cards
- Explicit selected-card Play/Discard controls
- Reduced always-on instruction text
